## 👥 角色设计（Characters）

### 角色特点
- **数量**: 1-2个主要角色
- **描述**: 简洁明了，突出视觉特征
- **风格**: Q版可爱，特征鲜明

### 示例角色

#### 主角：程序员小智
```json
{
  "name": "程序员小智",
  "description": "身穿蓝色T恤，戴眼镜，头发乱糟糟的，表情焦急，手里拿着笔记本电脑"
}
```
**视觉要点**:
- 服装：蓝色T恤（职业特征）
- 配饰：眼镜（程序员标志）
- 发型：乱糟糟（加班辛苦）
- 道具：笔记本电脑（工作必需）

#### 配角：程序小美
```json
{
  "name": "程序小美",
  "description": "穿着粉色连衣裙，长发飘飘，表情无辜，手里拿着一本《编程入门》"
}
```
**视觉要点**:
- 服装：粉色连衣裙（女性化）
- 发型：长发飘飘（可爱特征）
- 表情：无辜（反差萌）
- 道具：编程书（新手人设）

## 🎬 四格分镜结构（Panels）

### 第1格：开场
**目的**: 设定场景，引入问题
- **场景**: 程序员小智坐在电脑前，屏幕上显示着无数BUG，表情痛苦
- **动作**: 小智皱着眉头，用力敲打键盘
- **台词**: "今天怎么BUG这么多？"
- **视觉提示**: "程序员小智皱眉，电脑屏幕上密密麻麻的BUG，他正用力敲打键盘，背景是杂乱的编程书籍和饮料杯，色彩明亮，细节丰富。"

### 第2格：冲突/互动
**目的**: 引入第二个角色，增加互动
- **场景**: 程序小美手捧《编程入门》走过来，表情好奇
- **动作**: 小美眨巴着大眼睛，指着电脑屏幕
- **台词**: "小智，你今天怎么这么多BUG啊？"
- **视觉提示**: "程序小美手捧书籍，表情好奇，背景是一个温馨的编程环境，书架上摆放着各种编程书籍，小智坐在电脑前，屏幕上BUG依然众多。"

### 第3格：转折
**目的**: 引出主题，制造悬念
- **场景**: 小智看着小美，眼中闪过一丝疑惑
- **动作**: 小智疑惑地看着小美
- **台词**: "你不会是今天惊蛰吧？"
- **视觉提示**: "小智疑惑地看着小美，小美表情无辜，背景是一个充满编程氛围的房间，窗外的阳光透过窗户洒在两人身上，营造出温馨的氛围。"

### 第4格：结局/笑点
**目的**: 揭示答案，点题
- **场景**: 小美点头，然后突然笑出声
- **动作**: 小美突然笑出声，然后指着电脑屏幕
- **台词**: "是啊！惊蛰这天写代码，BUG自然就多了。"
- **视觉提示**: "小美笑得前俯后仰，手指向电脑屏幕，屏幕上BUG变得可爱起来，小智的表情从疑惑变为恍然大悟，背景是充满幽默感的编程场景。"

## 🎯 剧本创作要点

### 主题选择
✅ **好的主题**:
- 节日梗 + 职业场景（惊蛰+程序员）
- 日常共鸣 + 趣味反转
- 简单易懂 + 有记忆点

❌ **避免**:
- 过于复杂的背景设定
- 需要大量文字说明的梗
- 敏感或争议话题

### 对话设计
- **简短**: 每句不超过15字
- **口语化**: 自然对话风格
- **功能明确**: 推动剧情或制造笑点
- **有层次**: 问-答-转-合

### 视觉提示（visual_prompt）
**重要性**: ⭐⭐⭐⭐⭐

**必备要素**:
1. 角色状态（表情、动作）
2. 场景描述（环境、道具）
3. 色彩要求（明亮、Q版）
4. 细节补充（增加画面丰富度）

**公式**: `角色动作 + 主要元素 + 背景环境 + 风格要求`

**示例**:
```
"小美笑得前俯后仰，手指向电脑屏幕，
屏幕上BUG变得可爱起来，
小智的表情从疑惑变为恍然大悟，
背景是充满幽默感的编程场景。"
```

## 📋 剧本质量检查清单

### 结构完整性
- [ ] 标题简短有趣
- [ ] 有2个或更少主要角色
- [ ] 4格分镜完整
- [ ] 有明确的开场、发展、转折、结局

### 角色设计
- [ ] 角色特征鲜明（一眼能看出职业/性格）
- [ ] 角色外观描述具体（服装、道具、发型）
- [ ] 角色之间有对比或互补

### 对话质量
- [ ] 每句对话简短有力
- [ ] 对话推动剧情发展
- [ ] 有笑点或悬念
- [ ] 符合角色性格

### 视觉可执行性
- [ ] 每格都有详细的visual_prompt
- [ ] 场景描述清晰，AI可理解
- [ ] 包含风格要求（Q版、可爱、明亮）
- [ ] 避免过于复杂的构图

## 💡 生成提示词参考

### 剧本生成提示词
```
"{主题}，{关键元素}，{职业/场景}，
{风格要求}，{结构要求}"

示例：
"惊蛰，BUG，程序员，
搞笑，反转，可爱Q版风格，
简洁对话，适合漫画呈现"
```

### 角色生成提示词
```
"{角色类型}，{风格}，
{服装特征}，{发型}，{道具}，

示例：
"程序员，可爱Q版，
蓝色T恤，眼镜，头发乱糟糟"
```
